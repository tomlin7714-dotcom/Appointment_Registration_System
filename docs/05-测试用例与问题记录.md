## 二、开发中遇到的问题及解决方案

### 问题1：NumberSource.fee 类型不匹配

**描述：** 数据库 `number_source.fee` 字段类型为 `DECIMAL(10,2)`，但 Java 实体类使用 `Integer` 类型，导致金额丢失小数位。

**解决方案：** 将实体类 `fee` 字段类型从 `Integer` 改为 `Double`，同步更新所有 Controller 中的赋值逻辑和前端显示。

### 问题2：科室区域和诊室号未使用

**描述：** 数据库 `dept` 表已有 `area` 和 `room_number` 字段，但管理端 CRUD 和前端表格均未使用。

**解决方案：** 
- 后端 `DeptController` 增加 area/roomNumber 参数处理
- 新增 `GET /api/admin/dept/get` 接口用于编辑时获取单条数据
- 前端管理端增加区域和诊室号的显示与输入

### 问题3：排班时段格式不匹配

**描述：** 前端发送 `morning/afternoon/evening` 简写格式，后端期望 `"上午 08:00-10:00"` 完整格式。

**解决方案：** 后端增加格式转换逻辑，将简写映射为完整时间段，同时自动创建号源。

### 问题4：咨询表字段不一致

**描述：** 误以为 `consultation_form` 表的字段名为 `consultation_type` 等，实际为 `form_type`、`present_illness`、`past_history`。

**解决方案：** 重新对比数据库实际表结构，重写实体类、Mapper XML 和 Controller 中的字段映射。

### 问题5：AI导诊科室匹配错误

**描述：** "呼吸内科" 被错误匹配到 "内科"，因为 `"呼吸内科".contains("内科")` 为 true。

**解决方案：** 匹配逻辑改为按名称长度降序排列，长名称优先匹配。

### 问题6：头像上传路径问题

**描述：** Tomcat 将相对路径 `uploads/avatars` 解析到临时目录，导致上传失败。

**解决方案：** 改用 `System.getProperty("user.dir")` 获取项目根目录作为绝对路径基础。

### 问题7：微信图片显示问题

**描述：** 微信小程序 `<image>` 标签使用相对路径无法加载，需要完整 HTTP URL。

**解决方案：** 上传后在前端自动拼接 `apiBaseUrl` 前缀，同时从 API 加载时判断并补全。

### 问题8：微信端问诊历史状态显示错误

**描述：** 问诊历史状态显示为 "待就诊"，实际应显示预约状态。

**解决方案：** 将状态显示从 `detail.status`（表单状态）改为 `detail.appointmentStatus`（预约状态）。

### 问题9：医生端问诊单 doctorId 未设置

**描述：** 医生保存问诊单时未设置 `doctorId`，导致 JOIN 查不到医生信息。

**解决方案：** 在保存时通过预约→号源→排班链获取医生ID并设置。

### 问题10：管理端用户列表姓名/医生显示为 "-"

**描述：** `selectById` 仅查 `consultation_form` 基础字段，未 JOIN 关联表。

**解决方案：** 将 `selectById` 改为使用 `DetailResultMap`，加上 JOIN 和 `COALESCE` 备选逻辑。
