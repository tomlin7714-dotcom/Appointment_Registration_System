## 幻灯片 2：项目背景

### 传统挂号的痛点

| 问题 | 描述 |
|------|------|
| 🕐 排队时间长 | 挂号、缴费、就诊各环节均需排队 |
| 📋 号源不透明 | 不知道哪个医生还有号 |
| 🔄 流程繁琐 | 需要多次往返不同窗口 |
| 📊 信息不互通 | 就诊记录、诊断结果分散 |

### 解决方案

> 构建线上预约挂号平台，实现：
> - **患者**：随时随地预约，查看问诊记录
> - **医生**：灵活排班，高效叫号
> - **管理员**：统一管理用户/科室/医生

## 幻灯片 5：核心功能演示 — 预约挂号

### 预约流程（患者端）

```
① 首页 → ② 选科室 → ③ 选日期 → ④ 选医生
  → ⑤ 选时段 + 填问诊信息 → ⑥ 提交 → ⑦ 支付
```

### 关键点
- ✅ 多步引导式操作，清晰明了
- ✅ 实时显示号源余量
- ✅ 填写问诊信息（类型、主诉、病史）
- ✅ 支付后状态变为"待就诊"
- ✅ 支持取消预约（号源自动释放）

## 幻灯片 6：核心功能演示 — 诊间叫号与问诊

### 医生端流程

```
① 登录工作台 → ② 查看今日预约 → ③ 叫号
  → ④ 查看患者问诊信息 → ⑤ 填写诊断
  → ⑥ 保存 → 状态自动更新为"已完成"
```

### 关键点
- ✅ 语音播报叫号
- ✅ 显示患者自述（主诉、现病史等）
- ✅ 医生填写检查/诊断/治疗方案
- ✅ 保存后自动完成就诊
- ✅ 患者可在微信端查看诊断结果

## 幻灯片 11：总结与反思

### 项目成果

| 维度 | 成果 |
|------|------|
| ✅ 功能完整 | 覆盖预约、问诊、管理全流程 |
| ✅ 多端协同 | 小程序 + PC浏览器 |
| ✅ AI赋能 | 接入DeepSeek智能导诊 |
| ✅ 体验良好 | 多步引导、语音播报、状态实时更新 |

### 不足与改进

| 不足 | 改进方向 |
|------|---------|
| ❌ 密码明文存储 | 引入 BCrypt 加密 |
| ❌ 无微信支付对接 | 接入微信支付 SDK |
| ❌ 无短信通知 | 接入短信服务（就诊提醒） |
| ❌ 前端框架较基础 | 按课设要求改为 Vue3 + Unibest |
| ❌ 无单元测试 | 引入 JUnit + Mockito |
| ❌ 无日志框架集成 | 接入 ELK 日志系统 |

### 技术选型说明

> 课设要求使用 **Vue3 + Unibest**，但因时间紧迫，管理端和医生端采用了原生 HTML/JS，微信小程序采用原生框架。主要考虑：
> 1. 原生技术学习成本低，开发效率高
> 2. 项目功能复杂，减少框架调试时间
> 3. 核心是后端业务逻辑，前端框架不影响功能完整性
